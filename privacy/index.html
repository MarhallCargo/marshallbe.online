1. About This App

Marshall is a proxy client application — a network configuration utility for iOS. It allows users to import proxy server subscription links, view and manage server lists, check server availability, and establish encrypted network connections using the iOS NetworkExtension framework.

Marshall does not provide, operate, or control any proxy servers. The servers you connect to are determined entirely by the subscription link you provide. Marshall functions exclusively as a client interface, similar to other proxy utilities available on the App Store.

2. Core Principle

Marshall does not collect any personal data. All configuration data generated by your use of this application is stored entirely on your device. No user data is uploaded to any server operated by the Developer.

3. Data Stored on Your Device

The following data is stored locally on your device only and is never transmitted to the Developer:

3.1 Encrypted Storage (iOS Keychain)

  • Subscription URL — the HTTPS link you provide to import server configurations. Protected by iOS Keychain with kSecAttrAccessibleAfterFirstUnlock.
  • Server configuration cache — server names, addresses, ports, protocol parameters (VLESS, VMess, Trojan, Shadowsocks), and connection settings parsed from your subscription. Used to display the server list without re-downloading.

3.2 Local Preferences (UserDefaults)

  • Selected server ID — which server you last selected.
  • Language preference — your chosen interface language (Russian or English).
  • Auto-select setting — whether automatic best-server selection is enabled.

This data is sandboxed within the App and is automatically deleted when you uninstall Marshall.

4. Network Activity

Marshall performs the following network operations. All are initiated on your device; none involve the Developer's servers for tracking, analytics, or data collection:

4.1 Subscription Fetch

When you enter or refresh a subscription link, the App downloads server configuration data directly from the URL you provided. The request goes from your device to the subscription provider — the Developer has no visibility into this transaction.

4.2 Server Latency Checks

The App performs TCP connection tests to each server listed in your subscription to measure round-trip latency (ping). These are direct connections from your device to the respective servers. No data about the results is sent anywhere.

4.3 Service Availability Checks

The App can optionally check whether popular internet services (such as YouTube, Google, Instagram, Telegram, and others) are reachable from your current network by sending lightweight HTTP HEAD requests. These requests go directly from your device to the respective services. No results are collected or transmitted.

4.4 Network Tunnel

When you connect to a server, Marshall uses the iOS NetworkExtension framework and the open-source sing-box engine to create an encrypted tunnel. The tunnel configuration is built locally on your device and passed to the system Network Extension. All network traffic flows directly between your device and the server you selected — the Developer cannot see, intercept, or log any of this traffic.

4.5 Deep Link Import

Marshall supports the marshallvpn:// URL scheme, allowing other applications to pass a subscription URL to Marshall for import. The URL is processed entirely on your device.

5. Optional Feedback

Marshall includes an optional, in-app feedback form. If you choose to submit feedback:

  • Only the text you type is sent.
  • The submission is fully anonymous — no device identifiers, IP address, account information, subscription data, or server configuration is attached.
  • You are never required to use this feature.

6. Data We Do NOT Collect

The Developer does not collect, store, process, or have access to:

  • Personal identification information (name, email, phone number, account)
  • Device identifiers (IDFA, IDFV, serial number, hardware ID)
  • IP addresses or geographic location data
  • Browsing history, DNS queries, or network traffic content
  • Connection logs, session timestamps, durations, or bandwidth usage
  • App usage patterns, screen views, or behavioral analytics
  • Crash reports or diagnostic data

7. Third-Party Code and SDKs

Marshall does not include:

  • Advertising frameworks or ad tracking SDKs
  • Analytics services (Google Analytics, Firebase Analytics, Amplitude, Mixpanel, etc.)
  • Crash reporting SDKs (Crashlytics, Sentry, Bugsnag, etc.)
  • Social media SDKs or login frameworks
  • Attribution or marketing tracking SDKs

The only third-party component is the open-source sing-box library (Libbox), which runs entirely on your device and does not transmit any data to external services.

8. Third-Party Servers

Marshall is a client application. The proxy servers you connect to are provided by your subscription provider, not by the Developer. The Developer does not control, operate, or monitor these servers and bears no responsibility for their data handling practices, logging policies, or security measures. Please consult your subscription provider's privacy policy for information about how they handle your data.

9. Data Security

  • Sensitive data is stored in the iOS Keychain, Apple's hardware-backed encrypted storage.
  • Network tunnel configurations use modern encryption protocols as defined by sing-box.
  • The App enforces HTTPS for subscription links.
  • No user data exists on the Developer's servers — there is nothing to breach.

10. Children's Privacy

Marshall is not directed at children under the age of 13 (or the applicable minimum age in your jurisdiction). The Developer does not knowingly collect personal information from children. Since Marshall does not collect personal data from any user, no special age-related data processing occurs.

11. Your Rights and Data Control

  • Delete all data — uninstall the App from your device. All locally stored data (Keychain entries, preferences) is automatically removed.
  • Reset subscription — use the "Change subscription" button within the App to clear your current subscription and server data.
  • Remove VPN configuration — go to iOS Settings → General → VPN & Device Management to remove the network configuration profile.

This Privacy Policy may be updated to reflect changes in the App's functionality or applicable regulations. Significant changes will be communicated through an App update. The "Last updated" date at the top of this page indicates the most recent revision.

1. О приложении

Marshall — это прокси-клиент, утилита сетевой конфигурации для iOS. Приложение позволяет импортировать ссылки на серверные подписки, просматривать и управлять списком серверов, проверять доступность серверов и устанавливать зашифрованные сетевые соединения через фреймворк iOS NetworkExtension.

Marshall не предоставляет, не управляет и не контролирует какие-либо прокси-серверы. Серверы, к которым вы подключаетесь, полностью определяются предоставленной вами ссылкой подписки. Marshall работает исключительно как клиентский интерфейс, аналогично другим прокси-утилитам, доступным в App Store.

2. Основной принцип

Marshall не собирает персональные данные. Все конфигурационные данные, создаваемые при использовании приложения, хранятся исключительно на вашем устройстве. Никакие данные пользователей не загружаются на серверы Разработчика.

3. Данные на вашем устройстве

Следующие данные хранятся только локально на вашем устройстве и никогда не передаются Разработчику:

3.1 Зашифрованное хранилище (iOS Keychain)

  • URL подписки — HTTPS-ссылка для импорта конфигурации серверов. Защищена iOS Keychain с уровнем kSecAttrAccessibleAfterFirstUnlock.
  • Кэш конфигурации серверов — названия серверов, адреса, порты, параметры протоколов (VLESS, VMess, Trojan, Shadowsocks) и настройки подключения, полученные из подписки. Используется для отображения списка серверов без повторной загрузки.

3.2 Локальные настройки (UserDefaults)

  • ID выбранного сервера — какой сервер вы последний раз выбрали.
  • Язык интерфейса — выбранный язык (русский или английский).
  • Настройка автовыбора — включен ли автоматический выбор лучшего сервера.

Эти данные изолированы в песочнице приложения и автоматически удаляются при удалении Marshall.

4. Сетевая активность

Marshall выполняет следующие сетевые операции. Все они инициируются на вашем устройстве; ни одна не связана с серверами Разработчика для отслеживания, аналитики или сбора данных:

4.1 Загрузка подписки

При вводе или обновлении ссылки подписки приложение загружает конфигурацию серверов непосредственно с указанного вами URL. Запрос идёт с вашего устройства к провайдеру подписки — Разработчик не имеет доступа к этой транзакции.

4.2 Проверка задержки серверов

Приложение выполняет TCP-тесты соединения с каждым сервером из подписки для измерения задержки (пинга). Это прямые соединения между вашим устройством и серверами. Результаты никуда не отправляются.

4.3 Проверка доступности сервисов

Приложение может опционально проверять доступность популярных интернет-сервисов (YouTube, Google, Instagram, Telegram и др.) путём отправки HTTP HEAD-запросов. Запросы идут напрямую с вашего устройства. Результаты не собираются и не передаются.

4.4 Сетевой туннель

При подключении к серверу Marshall использует фреймворк iOS NetworkExtension и движок sing-box с открытым исходным кодом для создания зашифрованного туннеля. Конфигурация строится локально на устройстве и передаётся системному Network Extension. Весь трафик проходит напрямую между вашим устройством и выбранным сервером — Разработчик не может видеть, перехватывать или логировать этот трафик.

4.5 Импорт через Deep Link

Marshall поддерживает URL-схему marshallvpn://, позволяющую другим приложениям передавать ссылку подписки для импорта. URL обрабатывается полностью на вашем устройстве.

5. Опциональная обратная связь
